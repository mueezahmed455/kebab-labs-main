2 — Mock-First Adapter Pattern

Every external service is accessed through an adapter. The adapter checks for env vars at runtime — no env var = mock response. Swap in real keys and the same code goes live. No conditional logic in components.

3 — Key Interaction Flows

🛒 Order + Loyalty Points Flow

1
Customer places order
Cart → Checkout → Payment confirmed
2
Points auto-credited
DB trigger: 1pt per £1 spent → inserted into loyalty_points. If referral exists, referrer also gets 30pts.
3
Tier check
After insert, function checks total lifetime points → updates profile tier (Regular 0–199, Frequent 200–499, VIP 500+)
4
Redemption at checkout
Customer sees "Use 100 Lab Points (£1.00 off)" toggle. Applied as discount before payment intent is created.
5
Account dashboard
Points balance, tier badge, history of earn/spend events, progress bar to next tier

⚡ Flash Deal Flow

1
Admin creates deal
/admin/deals → set target (item/category/whole order), discount (% or £), start/end datetime
2
Deal goes live
Banner appears on homepage + /menu automatically. Countdown timer shows. Affected items show crossed-out original price.
3
Checkout applies discount
API validates deal still active on order creation. Applied as line item. Cannot stack with promo codes.
4
Deal expires
Banner disappears. Original prices restored. Scheduled deals auto-start/stop via cron or time check.

  • ComboBuilder — drag/select protein, base, 3 extras, sauce. Price reactive. Save to account or share.

  • ReferralPanel — unique link, copy + share, referral count, pending/paid credits
  • AnalyticsHeatmap — hour × day grid coloured by order volume
  • DealCreator — form with live preview of how the deal appears on the menu
  • ReviewCarousel — autoplay, pause on hover, star ratings, photo thumbnails
  • PWAInstallPrompt — bottom sheet on mobile after 3rd visit
5 — New API Routes (beyond base spec)
GET /api/loyalty/balance
Returns current user's points, tier, and earn history
POST /api/loyalty/redeem
Validates and reserves points for use in an order
GET /api/referral/link
Returns or creates the user's unique referral code + link
POST /api/referral/apply
Validates referral code on registration, credits both parties
GET /api/deals
Returns currently active flash deals (public)
POST /api/admin/deals
Create/update/delete flash deals (admin only)
POST /api/favourites/toggle
Add or remove a menu item from favourites
GET /api/favourites
Returns user's saved favourite items with full menu data
POST /api/subscribe
Email capture — validates email, inserts to email_subscribers, triggers welcome email
GET /api/social-proof
Returns today's order count, trending items. Cached 60s.
GET /api/wait-time
Returns current estimated wait (delivery + collection) based on admin's queue setting
POST /api/admin/wait-time
Admin sets queue load level: quiet / normal / busy
GET /api/admin/analytics/heatmap
Hour × day order volume for last 4 weeks
POST /api/push/subscribe
Saves PWA push subscription for a user
GET /sitemap.xml GET /robots.txt
SEO files — sitemap includes all menu item pages
